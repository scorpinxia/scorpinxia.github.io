树

树的出现

  • 大量输入 链表线性访问太慢 ，树大部分操作都是O(logN)，比较均衡
  • 二叉查找树：每种操作O(logN)

树的术语

root：根节点， 每个子树根r的child， r是每个子树的parent

  • N个点 N-1条边决定一个树
  • leaf：没有儿子的节点 siblings：相同父亲的是兄弟 grandparent grandchild类似
  • n1、n2……nk 的path 就是路径序列ni是ni+1的父亲 ，length就是路径经过的边 k-1 ，每个节点自己对自己有个length=0的路径，根到每个结点有唯一路径
  • n1 到n2 有路径,n1是n2的祖先ancestor,n2是n1后裔descendant。 n1!=n2 真祖先（proper ancestor) 真后裔(proper descendant)
  • ni的depth 深度：root到节点ni唯一路径的length
  • ni的height：ni到一片树叶的最长路径的长 leaf 的height就是0，因为没有儿子。

定理：一棵树的深度等于它最深的树叶的深度；该深度等于这个树的高

如上图 E的深度是1，高度是2；F深度1，高度1；Q深度3，高度0；

1.int recsum(int n) {
    if (n < 0)
        return 0;
    else if(n == 0 || n == 1)
        return 1;
    else
        return n + recsum(n - 1);
}

//尾递归
2.int fibonacci_tail(int n, int ret1, int ret2)
{
    if (n == 0)
        return ret1;
    else 
        return fibonacci_tail(n-1, ret2, ret1 + ret2);
}

尾递归可以用循环代替，转变后能克服递归的一些不足。例如：计算阶乘的递归计算过程属于线性递归，步骤数目的增长正比于输入n。也就是说，这个过程所需步骤的增长为O(n) ，空间需求的增长也为O(n) 。对于迭代的阶乘，步数还是O(n)而空间是O(1) ，也就是常数。

伸展树 （splay tree)

  • 保证从空树开始一系列M个的操作的时间最多O(MlogN),虽然有些操作还是O(N)，只要他很少就行；摊换时间概念
  • 基本想法：一个节点被访问后，它就要经过一系列AVL旋转被推到根，研究表明，一个节点被访问，它很可能不久就再被访问。伸展树不要求保留高度和平衡信息，因此它某种程度节省空间和简化代码

B树

我们现在可以存储整个数据结构到主存了，大O分析假设所有操作耗时相等 但是磁盘不一样。一台机器可以每秒几亿次指令，磁盘是机械运动，许多磁盘7200RPM，1分钟7200转，平均算转一半就要1/240秒，但是加上移动磁盘差不多也是1/120s，所以为了节省一次磁盘我们愿意进行大量运算。

假设查找驾驶记录1000w，每个关键词（名字32字节），记录256字节，不能都装入主存，二叉查找不行，不能低于logN。

B树特点 （M叉）

  • 数据项存储在树叶
  • 非叶结点存储直到M-1关键词指示搜索的方向。关键字i代表子树i+1的最小关键字。
  • 树的根或一个树叶，儿子数 2~M
  • 除根外，所有非树叶结点，儿子数【M/2】~M
  • 所有的树叶都在相同的深度，而且有【L/2】和L个数据项。

如下，所有非叶结点儿子数3和5之间（2到4关键词），根可能2儿子。L这时候取5 （不一定，每个结点代表一个磁盘区域

列如，一块可以8192字节，每个关键词32字节，M-1关键词 。M个分支，每个代表其他磁盘区域，设4字节。一个非叶结点内存需求 32M-32+4M ,不超过8192字节，最大M=228.
每个数据记录256字节，32个记录 刚好是一个块大小。所以 L=32。这样保证每片树叶16-32数据记录，每个内部结点（除根）至少114种方式分叉。1000w记录,至多1000w/16=625000树叶。最坏也在第四层，最坏情况 logM/2 N

实现TreeSet和TreeMAP

基本实现方式平衡二叉查找树，基本操作最坏对数，一般不用AVL树，而是自顶而下的红黑树。困难部分，迭代器支持。迭代器保留的是当前节点链接，困难时如何到下一个节点高效推进。方案如下：

  1. 构造迭代器时，让每个把包含TreeSet的数组存储。我们有toArray，不需要迭代器。
  2. 让迭代器保留存储通向当前节点路径的栈。但是这样迭代器有点大，导致迭代器代码臃肿
  3. 让查找树每个节点加上父节点地址。需要额外内存，迭代器代码臃肿。
  4. 让每个节点保留两个附加的链，一个通往更小的节点，一个通往更大的节点。占空间，迭代器简单。
  5. 只对那些具有null左链和null右链的节点保存附加的链。通过附加bool变量，线索树（threaded tree），用在许多平衡二叉查找。

    public static boolean oneCharOff(String word1,String word2){
        if(word1.length()!=word2.length()){
            return false;
        }

        int diff=0;
        for (int i = 0; i <word1.length() ; i++) {
            if(word1.charAt(i)!=word2.charAt(i)){
                diff++;
                if (diff>1){
                    return false;
                }
            }


        }

        return diff==1;
    }

    public static Map<String,List<String>> computeWords(Set<String> aWords){
        TreeMap<String, List<String>> adjWords = new TreeMap<>();
        String[] words=new String[aWords.size()];
         aWords.toArray(words);

        for (int i = 0; i < words.length; i++) {
            for (int j = i+1; j < words.length; j++) {

                if(oneCharOff(words[i],words[j])){
                    update(adjWords,words[i],words[j]);
                    update(adjWords,words[j],words[i]);
                }
            }
        }

        return adjWords;
    }

        update(aWords,wordsByLength);

        aWords.toArray(words);

                    }
                }
            }
        }

    }

    public static Map<String,List<String>> computeWords2(Set<String> aWords){
        TreeMap<String, List<String>> adjWords = new TreeMap<>();
        TreeMap<Integer, List<String>> wordsByLength = new TreeMap<>();

        update(aWords,wordsByLength);

        String[] words=new String[aWords.size()];
        aWords.toArray(words);

        for (Map.Entry<Integer, List<String>> everyEntry:wordsByLength.entrySet()) {

            List<String> groupsWords=everyEntry.getValue();
            int groupNum=everyEntry.getKey();

            for (int i = 0; i < groupNum; i++) {
                TreeMap<String, List<String>> repToWords = new TreeMap<>();

                for (String str:groupsWords
                     ) {
                    String rep=str.substring(0,i)+str.substring(i+1);
                    update(repToWords,rep,str);
                }

                for(List<String> wordCli:repToWords.values()){
                    if(wordCli.size()>=2){
                        for (String s1:wordCli) {
                            for (String s2:wordCli){
                                if (s1!=s2){
                                    update(adjWords,s1,s2);
                                }
                            }
                        }
                    }
                }
            }

        }
        return adjWords;
    }

    private static void update(Set<String> aWords, TreeMap<Integer, List<String>> wordsByLength) {
        for (String s:aWords){
            List<String> lengthList = wordsByLength.get(s.length());
            if(lengthList==null){
                lengthList=new ArrayList<>();
                wordsByLength.put(s.length(),lengthList);
            }
            lengthList.add(s);

        }
    }

            adjWords.put(word1,word1List);
        }

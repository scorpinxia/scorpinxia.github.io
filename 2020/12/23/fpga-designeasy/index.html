计数器架构

架构八步法

计数器规则

  • 1.初值、加一、结束值
  • 2.计数初值必须为0
  • 3.使用某计数值需要同时满足加1条件
    • +1 条件add_cnt&&cnt==4 表示计数到5个，而add_cnt==0&&cnt==4不表示技术到5个
  • 4.结束条件必须同时满足加1条件
  • 5.取某个数，assign 需要为(add_cnt&&cnt==num-1)
  • 6.结束时必须回到0
  • 7.限定范围 >= 和 <
    • 前8个 cnt>=0 && cnt<8
  • 8.设计步骤：先写计数器always,条件用名字替代，用assign写出加一条件 ,assign 写结束
    • always
      always @(posedge clk or posedge rst_n)begin  //状态寄存器在一个单独的always结构中
if (rst_n==1'b0)
        cnt <= 3'b0;
else if(add_cnt) begin
        if(end_cnt)
            cnt<=0;
        else
        cnt<=cnt+1;
end
end
    • assign +1 条件
      assign add_cnt= a==2;
    • assign 结束条件
      assign end_cnt=add_cnt&&cnt==10-1;
  • 9.加1条件必须与计数器严格对齐，其他信号一律向计数器对齐
    • 比如 有的到6拉高 有的到7拉高 不用6的来拉高7
  • 10.命名规范 add_ end_

状态机架构

状态机八步

状态机规则

  • 1.四段写法
    • 同步时序 always 格式化 次态迁移到现态
      always@(posedge clk or negedge rst_n)
begin
    if(!rst_n)
        state_cur <= IDLE;
    else begin
        state_cur <= state_next;
    end
end
    • 第二段 组合逻辑 always 描述状态转移条件 
      always@(*)
begin
        case(state_cur)
        IDLE:begin
            if(idle2s1_start)begin
                    state_next =S1;
                end
            else begin
                state_next = state_cur;
            end
        end
        
        S1:begin
            if(s12s2_start)begin
                    state_next =S2;
                end
            else begin
                state_next = state_cur;
            end
        end
        
        S2:begin
            if(s22idle_start)begin
                    state_next =IDLE;
                end
            else begin
                state_next = state_cur;
            end
        end
        
        default: begin
            state_next = IDLE;
        end
        endcase	
    end
end
    • 第三段 assign 定义转移条件。条件加上现态
      assign  idle2s1_start = state_cur==IDLE && 
assign  s12s2_start = state_cur==S1 && 
assign  s22idle_start = state_cur==S2 &&
    • 第四段 一个always设计一个信号 有多少个输出就有多少个always
  • 2.第一段写法不变
  • 3.第二代状态转移条件用信号 而不是具体
  • 4.assign xx2xx_start
  • 5.assign 加上当前状态
  • 6.状态不变 state_next=state_cur

FIFO架构

FIFO架构八步法

概述

FIFO规则

  • 1.Show-ahead 读模式： 一般两种 Show-ahead Normal
  • 2.读写隔离：读控制和写控制是独立的 （不能根据 读状态 读数据 决定写 ）
    • 例子：输入的报文保存FIFO，保存完整报文后，读出传送
    • din_eop收到完整报文，是用作读报文的开始吗？
    • 由于对第一个报文还没有处理，不能处理第二个报文
    • 数据要缓存后输出 ，din_eop也要缓存
    • 双FIFO架构
  • 3.读使能必须要判断空状态，并用组合逻辑产生 如上
    • Show-ahead模式 ,电路存在一定延时，a时刻FIFO以及是空的
    • 组合逻辑empty空 立马拉低
  • 4.处理报文，把指示信号和数据一起存入FIFO
    • 例子
    • 报文缓存格式
    • 读 最低位 次低位
  • 读写时钟不一样时必须用异步FIFO

Show-ahead 和Normal模式

  • Normal模式
    • 先有读使能，之后FIFO输出到数据总线，数据在下一个周期才有效
    • rdreq有效时 q还无效
  • show-ahead 模式
    • 先输出数据在总线上，没有rdreq 就一直输出D0

例子

  • 目标

  • 第一步 FIFO架构

    • 读写FIFO 都与输入的Din_eop有关，并且读出的数据与输入的报文长度n有关 ，不能让读和写有任何交互 长度也不行
    • 信息FIFO
      • 保存斜侧产生的，并由读测使用的数据
      • 信息FIFO的空标志 和保存个数统计 指示FIFO的报文个数
    • 数据FIFO
      • 保存输入数据 ，写数据din,din_eop 写使能 din_vld

  • 第二 读状态

    • 有些可以一直为1
    • 有些需要启动条件 rdl_flag (本次需要 存入一个完整报文才能开始读)

  • 第三 读状态启动和结束条件

    • 启动条件
      • 一般有 FIFO存够指定个数 （FIFO 保存够指定个数 、一个完整报文）
      • 保存完 完整报文
      • 其他特殊
    • 结束条件
      • 什么时候结束读FIFO 数据
      • 一种方法是计算读出来的长度，当读出来的长度等于写进长度，就代表结束点
      • 另一种读FIFO数据中的eop为1 表示到达结束点

  • 第四 信息FIFO读清条件

    • 信息FIFO作用

    • 保存写侧产生，并由读测使用的数据

    • 信息FIFO的空标志 和保存个数统计 ，指示数据FIFO报文的个数

    • 如果统一处理完整个报文读走 ，就无需考虑这个因素9shiy

    • 指示当前报文信息的

    • 读清条件就是eop_flag

  • 第五 生成FIFO IP核

    • 同读写时钟 同步FIFO

Graph Definition

  • G=(V,E) 由 vertex Edge组成
  • 顶点 :vertex
  • 边 Edge ,边 由 点对(v1,v2)决定 ，点对有序有向图。
  • 边可以 还有 weight
  • 路径 PATH v1、v2、……、vn length长 N-1
    • 环：顶点到顶点的边 路径0（特殊情况 ，一般讨论无环）
    • 简单路径 所以顶点互异（无环的，第一个节点可以和最后一个节点相同）
  • 圈：长度至少为1的路径 v1=vn (无向图的边需要互异)
  • DAG：有向无圈图 DAG （direction acyclic graph）
  • 连通的：无向图每一点到其他店都有一条路径
  • 强连通的: 有向图每一点到其他店都有一条路径
  • 弱连通的：有向图去掉方向每一点到其他店都有一条路径
  • 完全图：每一对顶点间都有存在一条边的图

贪婪算法

分阶段求解问题，每个阶段都把它当做最好的处理。但是不一定总是成功

找零钱，大部分人首先25美分、然后一角、5分、1分、 假设有12分币，要找15分的零钱
贪婪算法会选择 一个12分 三个一分，事实上1角加5分是最少的
  • 实现
    • 选择v1作为开始 开始找所有点的最小
    • 遍历v1和最近的v4
    • 遍历v2 (找邻接的unkown的点，计算路径 小于已知就不替换)选取v5、v3
    • 然后是v7、v6 所以都已知了

只要没有负边这个算法 总能实现 O(E+V^2)

  • 边是稠密的 算法基本最优
  • 边是O(V),算法就太慢了，可以用优先队列，顶点的选择是deleteMin
